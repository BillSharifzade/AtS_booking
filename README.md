# AtS Booking System

Простая система бронирования помещений для компании AtS:
- **Telegram-бот** для заказчиков (заявки, статусы, напоминания)
- **React-панель** для администраторов (просмотр, подтверждение, CRUD помещений)
- **FastAPI + PostgreSQL** на бэкенде

## Стек

| Слой       | Технологии                                         |
|------------|----------------------------------------------------|
| Бот        | aiogram 3.x, APScheduler (напоминания D-1 и H-1)   |
| API        | FastAPI, SQLAlchemy 2 async, Alembic, JWT          |
| БД         | PostgreSQL 16                                      |
| Web        | React 18 + Vite + TypeScript, чистый CSS           |
| Запуск     | docker compose (одна команда)                      |

## Первый запуск

```bash
cp .env.example .env      
docker compose up --build
```

Поднимутся четыре сервиса:
- `db` — Postgres (порт 5432, том `db_data`)
- `api` — FastAPI (порт 8000, авто-миграция при старте)
- `bot` — long-polling воркер
- `web` — Vite dev-сервер (порт 5173)

Откройте http://localhost:5173.

## Как войти в админку

1. Откройте http://localhost:5173 — увидите экран входа.
2. Введите свой Telegram ID. Узнать ID можно, написав боту `/whoami`.
3. Бот пришлёт 6-значный код. Введите его в форме. Готово.

Список ID администраторов берётся из `ADMIN_TELEGRAM_IDS` (через запятую). Сейчас разрешён только ID `1320166360`.

## Как сделать первое бронирование (после первого запуска)

1. Войдите в админку → раздел **Помещения** → создайте хотя бы одно помещение (название, зона A/Б/В, вместимость, рабочие часы).
2. В Telegram найдите бота и нажмите `/start`. Затем `/book` — пройдите шаги.
3. Заявка появится в админке в разделе **Заявки** со статусом «новая». Подтвердите её.
4. Заказчик получит уведомление, в группу `SAT_BOOKINGS_GROUP_CHAT_ID` уйдёт публикация.
5. За сутки и за час до начала бот отправит напоминание заказчику.

## Архитектура

```
server/                  ── общий Python-пакет для api и bot
  app/
    config.py            ── Settings из переменных окружения
    db.py                ── async-движок SQLAlchemy
    models.py            ── Room, Booking, StatusHistory, LoginCode, AuditLog
    schemas.py           ── Pydantic-модели для API
    security.py          ── JWT + хэш одноразовых кодов
    telegram.py          ── singleton aiogram Bot
    services/
      bookings.py        ── проверка пересечений, FSM статусов, аудит
      notifications.py   ── шаблоны сообщений в Telegram (RU)
    api/                 ── FastAPI: маршруты auth/rooms/bookings/audit
    bot/                 ── aiogram handlers, FSM бронирования, scheduler
  alembic/               ── миграции БД
frontend/                ── React-панель администратора (RU)
```

Бот и API делят один Docker-образ и работают с БД через те же SQLAlchemy-модели — никакого внутреннего HTTP между ними нет.

## Состояния заявки

```
new → processing → approved → completed → archived
                 → rejected → archived
```

Изменения статуса разрешены строго по этой диаграмме (`ALLOWED_TRANSITIONS` в `services/bookings.py`). Каждое изменение пишется в `status_history`, действия администратора — в `audit_log`.

## Бизнес-правила

- **Срочное бронирование** (< 2 суток до начала) помечается `is_urgent=true`, в уведомлениях админу появляется метка `(СРОЧНАЯ)`. Спецификация требует ручного контроля админа — это видимый сигнал, отказа не блокирует.
- **Пересечения**: запрос отклоняется, если в выбранном помещении уже есть заявка в статусах `new/processing/approved`, чьё время пересекается с запросом.
- **Рабочие часы помещения**: проверяются на каждом создании; событие должно начинаться и заканчиваться в один день.
- **Кофе-брейк**: булевый флаг + количество персон. Никакого меню в v1.
- **Уведомления**: заказчик получает DM на каждом переходе статуса; админы — DM на новые заявки; подтверждённые мероприятия публикуются в группу `SAT_BOOKINGS_GROUP_CHAT_ID`.
- **Напоминания**: `D-1` и `H-1` от `starts_at`, идемпотентно (флаги `reminder_day_sent`/`reminder_hour_sent`), сканер раз в 5 минут.

## Переменные окружения

См. `.env.example`. Ключевые:

| Переменная                 | Описание                                              |
|----------------------------|-------------------------------------------------------|
| `TELEGRAM_BOT_TOKEN`       | Токен бота (@BotFather)                               |
| `ADMIN_TELEGRAM_IDS`       | Список ID админов через запятую                       |
| `SAT_BOOKINGS_GROUP_CHAT_ID` | ID группы для публикации подтверждённых бронирований |
| `JWT_SECRET`               | Случайная длинная строка                              |
| `LOGIN_CODE_TTL_SECONDS`   | Срок жизни кода входа (по умолчанию 300)              |
| `DATABASE_URL`             | Авто-собирается из `POSTGRES_*` в docker-compose      |

## Полезные команды

```bash
# Логи отдельного сервиса
docker compose logs -f bot
docker compose logs -f api

# Войти в БД
docker compose exec db psql -U ats ats_booking

# Применить новую миграцию вручную (после правки моделей)
docker compose exec api alembic revision --autogenerate -m "описание"
docker compose exec api alembic upgrade head

# Пересобрать после правки requirements.txt / package.json
docker compose up --build
```
